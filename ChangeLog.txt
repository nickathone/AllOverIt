AllOverIt.Aws.Cdk.AppSync - v0.10.0
  * Moved the request/response mapping in v0.9.0 so the mapping type is (optionally) provided on the DataSource
    attribute. If a mapping type is not provided then the assumption is the registration will be manually provided
    in a MappingTemplates instance.
  * A mapping type defined on a DataSource can now support non-default constructors with the aid of a new
    MappingTypeFactory. The factory can be registered against exact types where the creator func is used to new up
    an instance of the required attribute with all required arguments. Overloads also exist that allow registration
    to be applied to a base class and the func can use a strategy against the provided type, or simple created by
    using Activator.CreateInstance(). The demo application shows how to do this.
